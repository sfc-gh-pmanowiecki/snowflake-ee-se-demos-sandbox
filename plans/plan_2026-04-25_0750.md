### Translation worker design (15 shared-pool agents)
Each worker loops:
1. `task_claim` next ready task
2. `web_fetch` Spanish source HTML
3. For large files: chunk for AI_COMPLETE token limits
4. Translate via `AI_COMPLETE('claude-opus-4-6', ...)` on `WH_AISQL_HOL`
5. Write to `aimlcases/<lang>/<target_filename>`
6. Mark task completed, claim next

### Translation rules
- **UI text** → target language (natural, fluent, proper grammar)
- **SQL identifiers** → English in all languages
- **SQL comments** → English
- **Internal links** → localized filenames for that language
- **`<html lang="xx">`** set correctly
- **Preserve:** HTML/CSS/JS structure, Snowflake terms, URLs, images

### Execution sequence
1. Size up `WH_AISQL_HOL` to LARGE
2. Create directories (`en/`, `pl/`, `es/`, `de/`, `fr/`)
3. Create team + 86 tasks (85 translations + 1 landing page)
4. Spawn 15 shared-pool translation workers + 1 landing page agent
5. Wait for completion
6. **Verification:** grep for Spanish leftovers, check internal links, validate landing page links
7. Remove old root-level `aimlcases/*.html` files
8. Size down warehouse to Medium
9. Commit & push
