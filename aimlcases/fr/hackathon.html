1 Le Concept

Un hackathon d'une demi-journée conçu pour que les équipes clients découvrent de première main comment résoudre des problèmes business en utilisant uniquement le langage naturel, sans avoir besoin de savoir programmer. La vedette est Cortex Code : l'assistant IA intégré à Snowsight qui transforme vos requêtes textuelles en code SQL, Python, ML et en tableaux de bord fonctionnels.

Proposition de valeur pour le client

  • Du zéro à la solution en quelques heures : D'une idée business à un pipeline ML + tableau de bord fonctionnel en une matinée
  • Sans barrière technique : Les profils business, produit, marketing et opérations peuvent construire des solutions IA
  • Données propres ou synthétiques : Chaque équipe choisit de travailler avec ses propres données ou celles du catalogue
  • Repartez avec votre solution : Tout ce qui est construit reste dans votre compte Snowflake, prêt à évoluer

Message clé : «Vous n'avez pas besoin d'une équipe data science pour résoudre des problèmes avec l'IA. Il suffit de savoir quoi demander.»

4 Dynamique du Hackathon

Formation des équipes

  • 3-4 personnes par équipe (idéal : mélange de profils business + technique)
  • Suggestion : mélanger les clients de différentes entreprises pour favoriser le networking
  • Chaque équipe choisit un nom créatif et un défi
  • Un mentor Snowflake assigné à chaque 2-3 équipes

Rôles suggérés dans l'équipe

  • Narrateur : Écrit les prompts en langage naturel et mène l'interaction avec Cortex Code
  • Architecte : Révise le code généré et décide de la structure des données
  • Designer : S'occupe du dashboard Streamlit et de l'expérience visuelle
  • Présentateur : Prépare la démo de 5 minutes pour les présentations finales

Comment ça fonctionne

  1. Chaque équipe reçoit un accès à un compte Snowflake trial (ou utilise le sien)
  2. Ils ouvrent le catalogue des cas d'usage et choisissent leur défi
  3. Ils suivent les prompts guidés comme base et les améliorent avec leurs propres idées
  4. Ils utilisent Cortex Code pour tout générer : données, ML, dashboard, automatisation
  5. À la fin, ils présentent leur solution à l'écran avec une démo en direct

Règle d'or : Tout est construit via Cortex Code en utilisant le langage naturel. Si quelqu'un veut écrire du SQL à la main, il peut — mais cela ne rapporte pas de points supplémentaires. Le but est de démontrer que l'IA le fait pour vous.

Gamification pendant l'événement

  • Réalisations éclair : Première équipe avec des données créées, premier modèle ML fonctionnel, premier dashboard déployé → bonus de points
  • Mur des prompts : Écran partagé où les équipes publient leurs meilleurs prompts pour inspirer les autres
  • Checkpoint à 11h15 : Tour rapide avant le café où chaque équipe partage son avancement en 30 secondes
  • Joker du mentor : Chaque mentor peut attribuer 1 «joker» à une équipe faisant preuve de créativité exceptionnelle

7 Questions Fréquentes

Les participants doivent-ils connaître SQL ou Python ?
Non. C'est justement l'objectif du hackathon : montrer qu'avec Cortex Code, vous pouvez construire des solutions AI/ML en utilisant uniquement le langage naturel. Les participants techniques pourront aller plus vite, mais ce n'est pas requis.
Combien d'équipes peuvent participer ?
Nous recommandons de 4 à 8 équipes (12 à 32 personnes). Moins de 4 équipes réduit la compétitivité ; plus de 8 complique les présentations dans le temps imparti.
Combien cela coûte-t-il en crédits Snowflake ?
Chaque équipe consommera de 5 à 15 $ de crédits avec des données synthétiques et un warehouse Small. Le compte trial gratuit inclut 400 $ de crédits, largement suffisant pour le hackathon et des semaines d'expérimentation ultérieure.
Que se passe-t-il si une équipe est bloquée ?
Les mentors Snowflake sont là pour aider. De plus, Cortex Code a la capacité de corriger ses propres erreurs : si quelque chose ne fonctionne pas, l'équipe peut coller l'erreur et demander sa correction. L'itération fait partie de l'apprentissage.
Peut-on utiliser les données réelles de l'entreprise ?
Oui, mais uniquement si l'équipe a l'autorisation interne de le faire. Les défis sont conçus avec des données synthétiques pour éviter les problèmes de confidentialité. Pour les équipes avancées souhaitant utiliser des données réelles, nous recommandons le « Défi Ouvert ».
Qu'emportent les participants après le hackathon ?
Tout ce qui est construit reste dans leur compte Snowflake : tables, modèles, dashboards et pipelines. Ils reçoivent également l'accès au catalogue complet des cas d'usage pour continuer à expérimenter avec d'autres défis d'autres secteurs.
Peut-on l'adapter à un après-midi ou à une journée complète ?
Absolument. Pour une journée complète, ajoutez un troisième sprint et des ateliers parallèles l'après-midi. Pour un demi après-midi, ajustez les sprints à 60 minutes chacun. La structure modulaire permet l'adaptation.
