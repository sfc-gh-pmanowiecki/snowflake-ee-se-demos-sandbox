1 Co to jest Cortex Code?

"Cortex Code to asystent programowania z wbudowaną sztuczną inteligencją w Snowsight, środowisku internetowym Snowflake. Umożliwia generowanie kodu SQL, Python i Streamlit bezpośrednio z języka naturalnego, przyspieszając rozwój rozwiązań analitycznych."

Kluczowe zdolności

  • Generowanie kodu SQL "— CREATE TABLE, INSERT, widoki, procedury, funkcje"
  • Cortex AI/ML — ML.CLASSIFICATION, ML.FORECAST, CORTEX.COMPLETE, CORTEX.SENTIMENT, Cortex Search
  • Feature Store "— Entity, FeatureView, get_features()"
  • Model Registry "— log_model(), get_model(), wersjonowanie"
  • Streamlit — Generowanie interaktywnych pulpitów nawigacyjnych
  • Zadania i potoki — Automatyzacja i orkiestracja

Ważne: "Cortex Code generuje kod, który jest uruchamiany w Twoim koncie Snowflake. Wszystko pozostaje w Twoim środowisku — bez zewnętrznych zależności, bez wysyłania danych na zewnątrz."

2 Wymagania wstępne

"Przed rozpoczęciem upewnij się, że masz skonfigurowane następujące elementy:"

Konto Snowflake

Ważne: Solicita a tu representante de Snowflake la creación de las cuentas necesarias para tu equipo. De esta forma aseguramos que están soportadas todas las funciones de Cortex Code y Cortex AI: ML.CLASSIFICATION, ML.FORECAST, CORTEX.COMPLETE, CORTEX.SENTIMENT, Cortex Search y Feature Store.

Dostęp do Cortex Code w Snowsight

"Po zalogowaniu się do swojego konta, po prawej stronie ekranu pojawi się panel kreatora" Cortex Code. Jest to Twój programista AI do uruchamiania wszystkich monity z tego katalogu:

Gotowy: "Jeśli zobaczysz panel Cortex Code z komunikatem „Jak mogę pomóc?” po prawej stronie, możesz zacząć kopiować i wklejać monity z dowolnego przypadku użycia."

Konto i uprawnienia

  • Konto Snowflake z dostępem do Snowsight
  • "Rola z uprawnieniami CREATE DATABASE, CREATE SCHEMA, CREATE TABLE"
  • "Warehouse aktywny (zalecany: rozmiar S dla rozwoju, M dla produkcji)"
  • Kod Cortex włączony na koncie (dostępny we wszystkich edycjach)

Dla funkcji zaawansowanych

  • Cortex AI/ML: "Wymaga, aby region obsługiwał funkcje Cortex (AWS us-west-2, us-east-1, eu-west-1 itd.)"
  • Sklep z funkcjami: Paquete snowflake-ml-python disponible
  • Rejestr modeli: Uprawnienia do tworzenia modeli w schemacie
  • Streamlit: Uprawnienie CREATE STREAMLIT w schemacie

Uwaga dotycząca regionów: Algunas funciones de Cortex AI (como ML.CLASSIFICATION o CORTEX.COMPLETE) requieren regiones específicas. Consulta la dokumentacja dostępności aby sprawdzić swój region.

3 Otwieranie Cortex Code w Snowsight

Cortex Code jest zintegrowany z wieloma powierzchniami Snowsight. Oto sposoby dostępu:

Opcja A — z Workspaces

  1. Wejdź do Snowsight → Projects → Workspaces
  2. Utwórz lub otwórz istniejące miejsce robocze
  3. Kliknij ikonę Cortex (✨) w prawym pasku bocznym lub użyj skrótu Cmd+Shift+Spacja (Mac) / Ctrl+Shift+Spacja Windows
  4. Zostanie otwarty panel Cortex Code po prawej stronie edytora

Tip: "Panel Cortex Code utrzymuje kontekst Twojego obszaru roboczego. Jeśli masz w edytorze odwołania do tabel, Cortex Code będzie je używał jako kontekstu do generowania bardziej precyzyjnego kodu."

4 Dodaj umiejętność Best-Practices do Cortex Code

"Aby poprawić wydajność i obniżyć koszty wygenerowanego kodu, możesz dodać skill de" best-practices a Cortex Code. Po załadowaniu będzie dostępny przed każdym monit i poprowadzi asystenta do generowania kodu bardziej wydajnego i zoptymalizowanego dla Snowflake.

4.1 — Pobieranie umiejętności

Pobierz skompresowany plik ze skillami najlepszych praktyk i zapisz go na swoim komputerze:

Pobierz best-practices.zip

4.2 — Rozpakuj lokalnie

"Rozpakuj plik, który pobrałeś na swój komputer. Otrzymasz folder o nazwie" best-practices z plikami umiejętności gotowymi do przesłania.

Tip: "Rozpakuj plik w łatwym do znalezienia miejscu (np. na pulpicie lub w folderze pobieranych plików), ponieważ w kolejnym kroku będziesz musiał wybrać go w Snowsight."

4.3 — Podnieś umiejętność Cortex Code w Snowsight

"W Snowsight, w panelu" Cortex Code", kliknij przycisk" +", wybierz opcję" «Upload Skill Folder(s)» i wybierz folder best-practices które właśnie rozpakowałeś lokalnie:

Uwaga: Przycisk + "znajduje się w górnej części panelu Cortex Code, obok selektora kontekstu. Po kliknięciu pojawi się menu — wybierz" «Upload Skill Folder(s)» "i przejdź do folderu, w którym rozpakowałeś skill."

4.4 – Użyj umiejętności przed każdym poleceniem

Po przesłaniu umiejętność będzie dostępna w Twojej sesji Cortex Code. Wpisz polecenie /best-practices "przed każdym poleceniem, aby ją włączyć i uzyskać bardziej efektywny kod:"

Gotowy: "Dzięki aktywnej umiejętności Cortex Code automatycznie zastosuje dobre praktyki Snowflake w całym generowanym kodzie: efektywne wykorzystanie magazynów, kluczy grupowania, optymalizacja zapytań i redukcja kosztów kredytów."

Struktura każdego przypadku użycia

Kliknij „Zobacz monity” na każdej karcie przypadku użycia zobaczysz:

  • Krok 1 — Konfiguracja środowiska: "Utwórz bazę danych, schemat i magazyn.Zawsze jest to pierwszy monit."
  • Krok 2-4 — Tworzenie danych: Generuje tabele z syntetycznymi danymi reprezentującymi domenę.
  • Krok 5-7 — Analiza i modelowanie: "Funkcje, modele ML, funkcje Cortex AI, Feature Store lub Model Registry."
  • Krok 8-9 — Dashboard: Kod Streamlit do interaktywnej wizualizacji.

Kolejność: Prompty są zaprojektowane do uruchamiania w kolejności. Każdy krok zależy od obiektów utworzonych w poprzednich krokach. Nie pomijaj kroków.

6 Wykonanie Przypadku Użytkowania Krok po Kroku

Zobaczmy cały proces na przykładzie praktycznym:

6.1 — Wybór branży i przypadku użycia

  1. Idź do katalog branż
  2. Wybierz branżę (np. Banca)
  3. "Przeglądaj karty lub użyj wyszukiwania, aby znaleźć przypadek użycia"
  4. Przeczytaj sekcję Problem aby zrozumieć ból biznesu

6.2 — Przeczytaj Poradnik

  1. Kliknij «Zobacz przewodnik»
  2. Przeczytaj: "Kontekst, Skupienie, Wyzwanie, Cele, Funkcjonalności, Dane, Stratyfikacja i Jak Używać"
  3. To da ci ramy koncepcyjne przed wygenerowaniem kodu

6.3 — Wykonaj monity

  1. Kliknij „Zobacz monity”
  2. W Snowsight otwórz workspace i aktywne Cortex Code
  3. Para cada paso:
    • Kliknij Kopiuj w monicie
    • Przyklej go do panelu Cortex Code
    • Cortex Code wygeneruje odpowiedni kod SQL/Python
    • Sprawdź to. przed wykonaniem (patrz sekcja 7)
    • Kliknij Run o Apply
    • "Sprawdź, czy został poprawnie wykonany"
    • Przejdź do następnego monitu
Ejemplo-- Prompt copiado del catálogo (Paso 1): "Crea la base de datos BANCA_FRAUDE_DB con esquema ANALYTICS y warehouse BANCA_WH (size S). Incluye comentarios del proyecto de detección de fraude transaccional." -- Cortex Code genera: CREATE DATABASE IF NOT EXISTS BANCA_FRAUDE_DB COMMENT = 'Proyecto de detección de fraude transaccional con Cortex AI/ML'; CREATE SCHEMA IF NOT EXISTS BANCA_FRAUDE_DB.ANALYTICS COMMENT = 'Schema principal para analytics de fraude'; CREATE WAREHOUSE IF NOT EXISTS BANCA_WH WAREHOUSE_SIZE = 'SMALL' AUTO_SUSPEND = 300 AUTO_RESUME = TRUE COMMENT = 'Warehouse para proyecto de fraude bancario'; USE DATABASE BANCA_FRAUDE_DB; USE SCHEMA ANALYTICS; USE WAREHOUSE BANCA_WH;

Oczekiwany rezultat: "Na koniec będziesz miał: bazę danych z tabelami danych syntetycznych, wytrenowane modele ML, skonfigurowane funkcje Cortex AI i działający pulpit nawigacyjny Streamlit."

7 Przegląd i walidacja wygenerowanego kodu

"Cortex Code generuje wysokiej jakości kod, ale" zawsze musisz to sprawdzić przed wykonaniem. Najważniejsze informacje:

Lista kontrolna

  • Nazwy przedmiotów: "Sprawdź, czy baza danych, schemat i magazyn są zgodne z Twoim środowiskiem"
  • Wielkość magazynu: "Dostosuj rozmiar do swojego obciążenia (S dla rozwoju, M-L dla produkcji)"
  • Ilość danych syntetycznych: Podpowiadane są reprezentatywne wolumeny; dostosuj je do potrzeb
  • Modele LLM: "Sprawdź, czy model odniesienia (np." llama3.1-70b, mistral-large2) jest dostępny w Twoim regionie
  • Funkcje ML: Confirma que ML.CLASSIFICATION, ML.FORECAST estén disponibles
  • Uprawnienia: Twoja rola musi mieć wymagane uprawnienia do obiektów

Dostosowanie do rzeczywistych danych

Wskazówki generują syntetyczne dane do demonstracji. Aby wprowadzić do produkcji:

  1. zastępuje CREATE TABLE ... INSERT syntetyczne CREATE VIEW o SELECT do twoich rzeczywistych tabel
  2. Dostosuj nazwy kolumn do swojego modelu danych
  3. Zmiana progów i parametrów biznesowych
  4. "Utrzymuj strukturę funkcji, modeli i pulpitów nawigacyjnych"

Dane syntetyczne: Wygenerowane dane są fikcyjne i reprezentatywne. Nie używać w rzeczywistych raportach biznesowych. Służą do walidacji potoku end-to-end przed podłączeniem rzeczywistych danych.

9 Automatyzacja za pomocą zadań i potoków

Ostatni monit w każdym przypadku użycia tworzy zadania Snowflake w celu zautomatyzowania potoku:

Co się automatyzuje

  • Przyjmowanie danych: "Okresowe pobieranie danych ze źródeł (codziennie, co tydzień)"
  • Aktualizacja funkcji: Ponowne obliczenie wskaźników i mierników
  • Scoring ML: Uruchomienie modeli na nowych danych
  • Ostrzeżenia: Powiadomienia o przekroczeniu progów
  • Przeszkolenie: Aktualizacja modeli okresowa

Typowa struktura zadań

SQL-- Task de scoring semanal CREATE OR REPLACE TASK scoring_semanal WAREHOUSE = MI_WH SCHEDULE = 'USING CRON 0 6 * * 1 Europe/Madrid' COMMENT = 'Scoring ML semanal sobre datos activos' AS CALL ejecutar_scoring(); -- Activar ALTER TASK scoring_semanal RESUME;

DAG zadań: "Zaawansowane przypadki użycia tworzą DAG (Directed Acyclic Graphs), gdzie Tasks zależą od innych. Cortex Code generuje zależności automatycznie z" AFTER.

10 Sprawdź wykorzystane wydatki

"Po uruchomieniu jednego lub kilku przypadków użycia użyj tego polecenia w Cortex Code, aby uzyskać skonsolidowane podsumowanie wszystkich kredytów i tokenów zużytych w bieżącym dniu: magazyny, funkcje Cortex AI, modele ML i Cortex Code w Snowsight."

Prompt — Podsumowanie kosztów dnia

"Skopiuj i wklej ten prompt bezpośrednio do Cortex Code (pamiętaj, aby aktywować" /best-practices wcześniej):

PromptGenera una consulta SQL que muestre un resumen consolidado de todos los costes y consumo de la cuenta Snowflake del día actual (CURRENT_DATE). Incluye las siguientes secciones: 1. CRÉDITOS DE WAREHOUSE - Fuente: SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY - Columnas: warehouse_name, SUM(credits_used) AS creditos_warehouse - Filtra por: DATE(start_time) = CURRENT_DATE() 2. FUNCIONES CORTEX AI (tokens y créditos) - Fuente: SNOWFLAKE.ACCOUNT_USAGE.CORTEX_FUNCTIONS_USAGE_HISTORY - Columnas: function_name, SUM(tokens_used) AS tokens_usados, SUM(credits_used) AS creditos_cortex - Filtra por: DATE(start_time) = CURRENT_DATE() 3. FUNCIONES ML SERVERLESS (ML.CLASSIFICATION, ML.FORECAST, etc.) - Fuente: SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY - Filtra por: service_type IN ('ML_FUNCTIONS','SNOWPARK_ML') AND DATE(usage_date) = CURRENT_DATE() - Columnas: service_type, credits_used_compute AS creditos_ml 4. CORTEX CODE EN SNOWSIGHT - Fuente: SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY - Filtra por: service_type = 'CORTEX_CODE' AND DATE(usage_date) = CURRENT_DATE() - Columnas: service_type, credits_used_compute AS creditos_cortex_code 5. RESUMEN TOTAL POR TIPO DE SERVICIO - Une todos los resultados anteriores con UNION ALL - Columnas finales: tipo_servicio, nombre, tokens_usados, creditos_gastados - Ordena por creditos_gastados DESC - Añade una fila TOTAL al final con SUM de todos los créditos Usa CTEs para cada sección. Si alguna vista no existe en la cuenta, sustítuyela por SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY filtrando por el service_type equivalente. Añade comentarios explicando cada CTE.

Uwaga: Dane dotyczące ACCOUNT_USAGE mają opóźnienie do 45 minut". Jeśli właśnie uruchomiłeś przypadek użycia i nie pojawiają się dane, poczekaj kilka minut i uruchom zapytanie ponownie."

Alternatywa: Si CORTEX_FUNCTIONS_USAGE_HISTORY "nie jest dostępny na Twoim koncie, Cortex Code automatycznie zastąpi tę sekcję przez" METERING_DAILY_HISTORY con service_type = 'AI_SERVICES'.

11 Porady i dobre praktyki

Desarrollo

  • Zawsze używaj /best-practices przed każdym poleceniem "— Aktywuj umiejętność, aby Cortex Code generował kod zoptymalizowany pod kątem wydajności i kosztów od pierwszego kroku"
  • Wykonaj monity Nie próbuj wykonać wszystkich monity naraz; każdy krok weryfikuje poprzedni
  • Nazwij swoje obiekty w sposób spójny — monity sugerują opisowe nazwy takie jak BANCA_FRAUDE_DB
  • Zapisz arkusz — Każdy przypadek użycia generuje sporo kodu; często zapisuje

Cortex Code

  • Bądź konkretny w monity "— Wskazówki katalogu są już szczegółowe, ale możesz dodać kontekst swojej firmy"
  • Iteruj po kodzie "Jeśli wygenerowany kod nie jest dokładnie tym, czego potrzebujesz, poproś Cortex Code, aby go dostosował"
  • Użyj kontekstu arkusza "— Jeśli w edytorze są już tabele, Cortex Code automatycznie je zreferuje"
  • "Połącz monity, jeśli potrzebujesz" "— dla zaawansowanych użytkowników, możesz połączyć 2-3 kroki w jednym dłuższym poleceniu"

Produkcja

  • Zastępuje syntetyczne dane rzeczywistymi "— Potok działa tak samo, zmieniasz tylko źródło danych"
  • Konfiguracja monitorów zasobów — Kontroluje zużycie kredytów magazynu
  • Sprawdź pozwolenia "— Używaj określonych ról dla każdego projektu (data_scientist, analyst, itp.)"
  • Monitoruj Zadania — Przejrzyj historię wykonania w Aktywność → Historia zadań
  • Wersjonuje modele Przypadki z Model Registry już zawierają wersjonowanie; użyj go do testów A/B

12 Często zadawane pytania

Czy do korzystania z monitów potrzebna jest znajomość SQL?
"Nie jest to konieczne, ale pomaga. Cortex Code generuje SQL za Ciebie, ale zrozumienie podstawowych pojęć (CREATE TABLE, SELECT, JOIN) pozwoli Ci z pewnością przeglądać i dostosowywać kod."
Czy syntetyczne dane zużywają dużo kredytów?
"Minimum. Generowanie danych syntetycznych za pomocą INSERT zużywa bardzo mało kredytów (sekund magazynu S). Największe zużycie pochodzi z treningu ML i funkcji Cortex, które zależą od objętości danych."
Czy mogę używać monity z moimi prawdziwymi danymi od początku?
"Tak. Zastąp kroki „Tworzenie danych” odniesieniami do istniejących tabel. Upewnij się, że nazwy kolumn są zgodne z tym, czego oczekują kolejne kroki, lub poproś Cortex Code o dostosowanie kodu."
"Co zrobić, jeśli monit wygeneruje błąd?"
"Najczęstsze: (1) brak wykonania poprzedniego kroku, który tworzy odwoływaną tabelę, (2) magazyn nie jest aktywny, (3) funkcja Cortex nie jest dostępna w Twoim regionie. Sprawdź komunikat o błędzie i poproś Cortex Code, aby go poprawił: „Ten kod powoduje błąd X, popraw go”."
Ile czasu zajmuje ukończenie przypadku użycia?
"Od 8 do 15 minut na wykonanie wszystkich wywołań z syntetycznymi danymi. Czas podany na każdej karcie jest realistyczną oceną, w tym przegląd kodu."
Czy mogę łączyć przypadki użycia z różnych branż?
"Absolutnie. Wiele wzorców jest przekrojowych: sklep z funkcjami klienta działa tak samo w bankowości, handlu detalicznym czy telekomunikacji. Dostosuj nazwy kolumn i logikę biznesową."
Czy Cortex Code zapamiętuje poprzednie kroki?
"W ramach tej samej sesji arkusza kalkulacyjnego, tak. Jeśli zamkniesz i ponownie otworzysz, musisz nadać mu kontekst. Dlatego zalecamy wykonanie wszystkich kroków przypadku użycia w jednej ciągłej sesji."
Czy mogę zmienić monity?
"Tak, i jest to zalecane. Monity są zaprojektowane jako punkt wyjścia. Dodaj szczegóły dotyczące swojej firmy, dostosuj wolumeny danych, zmień progi lub dodaj kolumny specyficzne dla Twojej domeny, aby uzyskać bardziej precyzyjne wyniki."
Jaka jest różnica między „Zobacz przewodnik” a „Zobacz monity”?
Zobacz przewodnik "przedstawia ramy koncepcyjne: kontekst problemu, cele, dane, których potrzeba, oraz strategię wdrożenia." Zobacz monity "zawiera monity krok po kroku, które kopiujesz i wklejasz bezpośrednio do Cortex Code, aby wygenerować kod."
Czy muszę coś zainstalować?
"Prawie nic. Cały kod jest wykonywany w Snowsight w przeglądarce, bez środowisk Python ani narzędzi zewnętrznych. Jedynym krokiem wstępnym jest" pobierz umiejętność najlepszych praktyk plik best-practices.zip"), rozpakuj go na swoim komputerze i załaduj folder do Cortex Code z przycisku" + w Snowsight. Po przesłaniu będzie dostępny za pomocą polecenia /best-practices i nie będziesz musiał tego robić ponownie.
