1 ¿Qué es Cortex Code?

Cortex Code to asystent programowania z AI zintegrowany w Snowsight, środowisku webowym Snowflake. Umożliwia generowanie kodu SQL, Python i Streamlit bezpośrednio z języka naturalnego, przyspieszając rozwój rozwiązań analitycznych.

Główne możliwości

  • Generowanie kodu SQL — CREATE TABLE, INSERT, vistas, procedures, funciones
  • Cortex AI/ML — ML.CLASSIFICATION, ML.FORECAST, CORTEX.COMPLETE, CORTEX.SENTIMENT, Cortex Search
  • Feature Store — Entity, FeatureView, get_features()
  • Model Registry — log_model(), get_model(), versionado
  • Streamlit — Generación de dashboards interactivos
  • Tasks & Pipelines — Automatización y orquestación

Ważne: Cortex Code genera código que se ejecuta en tu cuenta de Snowflake. Todo queda dentro de tu entorno — sin dependencias externas, sin enviar datos fuera.

2 Requisitos Previos

Przed rozpoczęciem upewnij się, że masz skonfigurowane następujące elementy:

Konto Snowflake

Ważne: Solicita a tu representante de Snowflake la creación de las cuentas necesarias para tu equipo. De esta forma aseguramos que están soportadas todas las funciones de Cortex Code y Cortex AI: ML.CLASSIFICATION, ML.FORECAST, CORTEX.COMPLETE, CORTEX.SENTIMENT, Cortex Search y Feature Store.

Dostęp do Cortex Code w Snowsight

Una vez dentro de tu cuenta, en la parte derecha de la pantalla aparece el panel del asistente de Cortex Code. To Twój partner programistyczny AI do wykonywania wszystkich promptów z tego katalogu:

Gotowe: Si ves el panel de Cortex Code con el mensaje «How can I help?» a la derecha, ya puedes empezar a copiar y pegar los prompts de cualquier caso de uso.

Konto i uprawnienia

  • Konto Snowflake z dostępem do Snowsight
  • Rola z uprawnieniami CREATE DATABASE, CREATE SCHEMA, CREATE TABLE
  • Aktywny warehouse (zalecane: size S dla rozwoju, M dla produkcji)
  • Cortex Code włączony na koncie (dostępne we wszystkich edycjach)

Dla zaawansowanych funkcji

  • Cortex AI/ML: Requiere que la región soporte Cortex Functions (AWS us-west-2, us-east-1, eu-west-1, etc.)
  • Feature Store: Paquete snowflake-ml-python disponible
  • Model Registry: Permisos para crear modelos en el esquema
  • Streamlit: Permiso CREATE STREAMLIT en el esquema

Uwaga dotycząca regionów: Algunas funciones de Cortex AI (como ML.CLASSIFICATION o CORTEX.COMPLETE) requieren regiones específicas. Consulta la dokumentacja dostępności para verificar tu región.

3 Abrir Cortex Code en Snowsight

Cortex Code jest zintegrowany w różnych powierzchniach Snowsight. Oto sposoby dostępu:

Opcja A — Z Workspaces

  1. Entra en Snowsight → Projects → Workspaces
  2. Utwórz lub otwórz istniejący workspace
  3. Haz clic en el icono de Cortex (✨) na prawym pasku bocznym, lub użyj skrótu Cmd+Shift+Space (Mac) / Ctrl+Shift+Space (Windows)
  4. Po prawej stronie edytora otworzy się panel Cortex Code

Tip: El panel de Cortex Code mantiene el contexto de tu workspace. Si tienes tablas referenciadas en el editor, Cortex Code las usará como contexto para generar código más preciso.

4 Añadir la Skill de Best-Practices a Cortex Code

Para mejorar el rendimiento y reducir costes del código generado, puedes añadir la skill de best-practices a Cortex Code. Una vez cargada, estará disponible antes de cada prompt y orientará al asistente a generar código más eficiente y optimizado para Snowflake.

4.1 — Pobierz skill

Pobierz skompresowany plik ze skill best-practices i zapisz go na swoim komputerze:

Pobierz best-practices.zip

4.2 — Rozpakuj lokalnie

Descomprime el fichero descargado en tu equipo. Obtendrás una carpeta llamada best-practices con los archivos de la skill listos para subir.

Tip: Descomprime en un lugar fácil de localizar (por ejemplo, el Escritorio o tu carpeta de Descargas), ya que en el siguiente paso necesitarás seleccionarla desde Snowsight.

4.3 — Prześlij skill do Cortex Code w Snowsight

En Snowsight, dentro del panel de Cortex Code, haz clic en el botón +, selecciona la opción «Upload Skill Folder(s)» i wybierz folder best-practices que acabas de descomprimir localmente:

Uwaga: Przycisk + znajduje się w górnej części panelu Cortex Code, obok selektora kontekstu. Po kliknięciu pojawi się menu — wybierz «Upload Skill Folder(s)» y navega hasta donde descomprimiste la skill.

4.4 — Użyj skill przed każdym promptem

Una vez subida, la skill quedará disponible en tu sesión de Cortex Code. Escribe el comando /best-practices antes de cada prompt para activarla y obtener código más eficiente:

Gotowe: Con la skill activa, Cortex Code aplicará automáticamente buenas prácticas de Snowflake en todo el código generado: uso eficiente de warehouses, clustering keys, optimización de consultas y reducción de costes de créditos.

Struktura każdego przypadku użycia

Al hacer clic en «Zobacz Prompty» en cualquier tarjeta de caso de uso, verás:

  • Krok 1 — Konfiguracja środowiska: Crea la base de datos, esquema y warehouse. Siempre es el primer prompt.
  • Kroki 2-4 — Tworzenie danych: Genera tablas con datos sintéticos representativos del dominio.
  • Kroki 5-7 — Analiza i modelowanie: Features, modelos ML, funciones Cortex AI, Feature Store o Model Registry.
  • Krok 8-9 — Dashboard: Código Streamlit para visualización interactiva.
  • Ostatni krok — Pipeline: Tasks para automatización y operación continua.

Sekwencyjność: Los prompts están diseñados para ejecutarse en orden. Cada paso depende de los objetos creados en los pasos anteriores. No saltes pasos.

6 Ejecutar un Caso de Uso Paso a Paso

Zobaczmy cały proces na praktycznym przykładzie:

6.1 — Wybór branży i przypadku użycia

  1. Ve al katalog branż
  2. Elige una industria (por ejemplo, Banca)
  3. Przeglądaj karty lub użyj wyszukiwarki, aby znaleźć przypadek użycia
  4. Lee la sección «Problem» para entender el dolor de negocio

6.2 — Przeczytaj Przewodnik

  1. Haz clic en «Zobacz Przewodnik»
  2. Lee las secciones: Kontekst, Fokus, Wyzwanie, Cele, Funkcjonalności, Dane, Stratyfikacja i Jak Używać
  3. To da ci ramy koncepcyjne przed wygenerowaniem kodu

6.3 — Wykonaj Prompty

  1. Haz clic en «Zobacz Prompty»
  2. En Snowsight, abre un workspace i aktywuj Cortex Code
  3. Para cada paso:
    • Haz clic en «Kopiuj» en el prompt
    • Wklej go w panelu Cortex Code
    • Cortex Code wygeneruje odpowiedni kod SQL/Python
    • Sprawdź to antes de ejecutar (ver sección 7)
    • Haz clic en «Run» o «Apply»
    • Sprawdź, czy wykonano poprawnie
    • Przejdź do następnego promptu
Ejemplo-- Prompt copiado del catálogo (Paso 1): "Crea la base de datos BANCA_FRAUDE_DB con esquema ANALYTICS y warehouse BANCA_WH (size S). Incluye comentarios del proyecto de detección de fraude transaccional." -- Cortex Code genera: CREATE DATABASE IF NOT EXISTS BANCA_FRAUDE_DB COMMENT = 'Proyecto de detección de fraude transaccional con Cortex AI/ML'; CREATE SCHEMA IF NOT EXISTS BANCA_FRAUDE_DB.ANALYTICS COMMENT = 'Schema principal para analytics de fraude'; CREATE WAREHOUSE IF NOT EXISTS BANCA_WH WAREHOUSE_SIZE = 'SMALL' AUTO_SUSPEND = 300 AUTO_RESUME = TRUE COMMENT = 'Warehouse para proyecto de fraude bancario'; USE DATABASE BANCA_FRAUDE_DB; USE SCHEMA ANALYTICS; USE WAREHOUSE BANCA_WH;

Oczekiwany wynik: Al final de todos los pasos tendrás: base de datos con tablas de datos sintéticos, modelos ML entrenados, funciones Cortex AI configuradas y un dashboard Streamlit funcional.

7 Revisar y Validar el Código Generado

Cortex Code genera código de alta calidad, pero zawsze musisz to sprawdzić antes de ejecutar. Puntos clave:

Checklist przeglądu

  • Nazwy obiektów: Verifica que la base de datos, esquema y warehouse coincidan con tu entorno
  • Warehouse size: Ajusta el tamaño según tu carga (S para desarrollo, M-L para producción)
  • Wolumen danych syntetycznych: Los prompts sugieren volúmenes representativos; ajústalos según necesidad
  • Modele LLM: Zweryfikuj, że referencyjny model (np. llama3.1-70b, mistral-large2) był dostępny w Twoim regionie
  • Funkcje ML: Confirma que ML.CLASSIFICATION, ML.FORECAST estén disponibles
  • Uprawnienia: Tu rol debe tener los grants necesarios sobre los objetos

Dostosuj do rzeczywistych danych

Prompty generują dane syntetyczne do celów demonstracyjnych. Aby wdrożyć na produkcję:

  1. Sustituye los CREATE TABLE ... INSERT syntetycznych przez CREATE VIEW o SELECT sobre tus tablas reales
  2. Dostosuj nazwy kolumn do swojego modelu danych
  3. Zmodyfikuj progi i parametry biznesowe
  4. Zachowaj strukturę features, modeli i dashboardów

Dane syntetyczne: Los datos generados son ficticios y representativos. No usar en reportes de negocio reales. Sirven para validar el pipeline end-to-end antes de conectar datos reales.

9 Automatizar con Tasks y Pipelines

Ostatni prompt każdego przypadku użycia tworzy Tasks w Snowflake do automatyzacji pipeline:

Co jest automatyzowane

  • Ingestion danych: Carga periódica desde fuentes (diaria, semanal)
  • Aktualizacja features: Recalcular indicadores y métricas
  • Scoring ML: Ejecutar modelos sobre nuevos datos
  • Alerty: Notificaciones cuando se superan umbrales
  • Ponowne trenowanie: Actualizar modelos periódicamente

Typowa struktura Tasks

SQL-- Task de scoring semanal CREATE OR REPLACE TASK scoring_semanal WAREHOUSE = MI_WH SCHEDULE = 'USING CRON 0 6 * * 1 Europe/Madrid' COMMENT = 'Scoring ML semanal sobre datos activos' AS CALL ejecutar_scoring(); -- Activar ALTER TASK scoring_semanal RESUME;

DAG z Tasks: Zaawansowane przypadki użycia tworzą DAGs (Directed Acyclic Graphs), gdzie jedne Tasks zależą od innych. Cortex Code automatycznie generuje zależności za pomocą AFTER.

10 Comprobar Gasto Utilizado

Po wykonaniu jednego lub kilku przypadków użycia, użyj tego promptu w Cortex Code, aby uzyskać skonsolidowane podsumowanie wszystkich kredytów i tokenów zużytych w bieżącym dniu: warehouses, funkcje Cortex AI, modele ML i Cortex Code w Snowsight.

Prompt — Podsumowanie kosztów dnia

Copia y pega este prompt directamente en Cortex Code (recuerda activar /best-practices antes):

PromptGenera una consulta SQL que muestre un resumen consolidado de todos los costes y consumo de la cuenta Snowflake del día actual (CURRENT_DATE). Incluye las siguientes secciones: 1. CRÉDITOS DE WAREHOUSE - Fuente: SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY - Columnas: warehouse_name, SUM(credits_used) AS creditos_warehouse - Filtra por: DATE(start_time) = CURRENT_DATE() 2. FUNCIONES CORTEX AI (tokens y créditos) - Fuente: SNOWFLAKE.ACCOUNT_USAGE.CORTEX_FUNCTIONS_USAGE_HISTORY - Columnas: function_name, SUM(tokens_used) AS tokens_usados, SUM(credits_used) AS creditos_cortex - Filtra por: DATE(start_time) = CURRENT_DATE() 3. FUNCIONES ML SERVERLESS (ML.CLASSIFICATION, ML.FORECAST, etc.) - Fuente: SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY - Filtra por: service_type IN ('ML_FUNCTIONS','SNOWPARK_ML') AND DATE(usage_date) = CURRENT_DATE() - Columnas: service_type, credits_used_compute AS creditos_ml 4. CORTEX CODE EN SNOWSIGHT - Fuente: SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY - Filtra por: service_type = 'CORTEX_CODE' AND DATE(usage_date) = CURRENT_DATE() - Columnas: service_type, credits_used_compute AS creditos_cortex_code 5. RESUMEN TOTAL POR TIPO DE SERVICIO - Une todos los resultados anteriores con UNION ALL - Columnas finales: tipo_servicio, nombre, tokens_usados, creditos_gastados - Ordena por creditos_gastados DESC - Añade una fila TOTAL al final con SUM de todos los créditos Usa CTEs para cada sección. Si alguna vista no existe en la cuenta, sustítuyela por SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY filtrando por el service_type equivalente. Añade comentarios explicando cada CTE.

Uwaga: Dane z ACCOUNT_USAGE mają opóźnienie do 45 minut. Jeśli właśnie wykonałeś przypadek użycia i dane się nie pojawiają, poczekaj kilka minut i ponownie uruchom zapytanie.

Alternatywa: Si CORTEX_FUNCTIONS_USAGE_HISTORY nie jest dostępny na Twoim koncie, Cortex Code automatycznie zastąpi tę sekcję przez METERING_DAILY_HISTORY con service_type = 'AI_SERVICES'.

11 Consejos y Buenas Prácticas

Desarrollo

  • Usa siempre /best-practices antes de cada prompt — Activa la skill para que Cortex Code genere código optimizado en rendimiento y costes desde el primer paso
  • Wykonuj prompt po prompcie — No intentes ejecutar todos los prompts de golpe; cada paso valida el anterior
  • Nazywaj swoje obiekty konsekwentnie — Prompty sugerują opisowe nazwy takie jak BANCA_FRAUDE_DB
  • Zapisz worksheet — Cada caso de uso genera bastante código; guarda frecuentemente

Cortex Code

  • Bądź konkretny w promptach — Los prompts del catálogo ya son detallados, pero puedes añadir contexto de tu negocio
  • Iteruj nad kodem — Si el código generado no es exactamente lo que necesitas, pide a Cortex Code que lo ajuste
  • Użyj kontekstu arkusza roboczego — Si ya tienes tablas en el editor, Cortex Code las referenciará automáticamente
  • Łącz prompty, jeśli potrzebujesz — Para usuarios avanzados, puedes combinar 2-3 pasos en un solo prompt más largo

Produkcja

  • Zastąp dane syntetyczne rzeczywistymi — El pipeline funciona igual, solo cambias la fuente de datos
  • Skonfiguruj Resource Monitors — Controla el consumo de créditos del warehouse
  • Sprawdź uprawnienia — Usa roles específicos para cada proyecto (data_scientist, analyst, etc.)
  • Monitoruj Tasks — Sprawdź historię wykonania w Activity → Task History
  • Wersjonuj modele — Los casos con Model Registry ya incluyen versionado; úsalo para A/B testing

12 Preguntas Frecuentes

Czy muszę znać SQL, żeby używać promptów?
Nie jest niezbędna, ale pomaga. Cortex Code generuje SQL za Ciebie, ale zrozumienie podstawowych pojęć (CREATE TABLE, SELECT, JOIN) pozwoli Ci przeglądać i dostosowywać kod z pewnością.
Czy dane syntetyczne zużywają dużo kredytów?
Minimalne. Generowanie danych syntetycznych z INSERT zużywa bardzo mało kredytów (sekundy warehouse S). Największe zużycie pochodzi z trenowania ML i funkcji Cortex, które zależą od wolumenu danych.
Czy mogę używać promptów z moimi rzeczywistymi danymi od początku?
Tak. Zastąp kroki «Utwórz dane» odwołaniami do istniejących tabel. Upewnij się, że nazwy kolumn odpowiadają temu, czego oczekują kolejne kroki, lub poproś Cortex Code o dostosowanie kodu.
Co zrobić, jeśli prompt generuje błąd?
Najczęstsze przyczyny: (1) brak wykonania poprzedniego kroku tworzącego przywoływaną tabelę, (2) warehouse nie jest aktywny, (3) funkcja Cortex nie jest dostępna w Twoim regionie. Sprawdź komunikat błędu i poproś Cortex Code o poprawę: «Ten kod zwraca błąd X, popraw go».
Ile czasu zajmuje ukończenie przypadku użycia?
Od 8 do 15 minut na wykonanie wszystkich promptów z danymi syntetycznymi. Czas wskazany na każdej karcie to realistyczne oszacowanie uwzględniające przegląd kodu.
Czy mogę łączyć przypadki użycia z różnych branż?
Absolutnie. Wiele wzorców jest uniwersalnych: Feature Store klienta działa tak samo w Bankowości, Retail czy Telekomunikacji. Dostosuj nazwy kolumn i logikę biznesową.
Czy Cortex Code pamięta poprzednie kroki?
W ramach tej samej sesji worksheet, tak. Jeśli zamkniesz i otworzysz ponownie, będziesz musiał podać kontekst. Dlatego zalecamy wykonanie wszystkich kroków przypadku użycia w jednej ciągłej sesji.
Czy mogę modyfikować prompty?
Tak, i jest to zalecane. Prompty są zaprojektowane jako punkt wyjścia. Dodaj szczegóły swojego biznesu, dostosuj wolumeny danych, zmień progi lub dodaj kolumny specyficzne dla Twojej domeny, aby uzyskać dokładniejsze wyniki.
Jaka jest różnica między „Zobacz Przewodnik"" a „Zobacz Prompty""?
Zobacz Przewodnik przedstawia ramy koncepcyjne: kontekst problemu, cele, wymagane dane i strategię wdrożenia. Zobacz Prompty contiene los prompts paso a paso que copias y pegas directamente en Cortex Code para generar el código.
Czy muszę coś zainstalować?
Casi nada. Todo el código se ejecuta dentro de Snowsight en el navegador, sin entornos Python ni herramientas externas. El único paso previo es pobierz skill best-practices (plik best-practices.zip), descomprimirla en tu equipo y subir la carpeta a Cortex Code desde el botón + w Snowsight. Po przesłaniu będzie dostępny za pomocą polecenia /best-practices y no necesitarás repetir el proceso.
